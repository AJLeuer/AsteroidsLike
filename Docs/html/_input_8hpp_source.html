357 /**
358  * SwitchableKeyInputRegister is a variant of KeyInputRegister that can hold multiple callback functions
359  * (and members to call those functions on), as well as several different keyboard keys at the same time.
360  * This is in contrast to the single function pointer held in any KeyInputRegister. The real advantage of SwitchableKeyInputRegister
361  * is its ability to choose different callbacks based on a descending order of priority that is set when the SwitchableKeyInputRegister
362  * is created. A SwitchableKeyInputRegister is part of a linked-list of pointers to KeyInputRegisters or other SwitchableKeyInputRegisters,
363  * and each one is able to "shut off" subsequent members in its list from providing their callback functions. The list should be initialized
364  * with the highest priority KeyInputRegisters at the front, with the fallback, lower priority ones following. Any KeyInputRegister or
365  * SwitchableKeyInputRegister's callback function will only be called if callback function belonging to the KeyInputRegister prior to it
366  * in the list was not.
367  */
